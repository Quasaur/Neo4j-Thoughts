## 🔧 Proposed Fix Plan

### Fix 1 — Remove Misplaced Semicolons (7 files)

**Problem:** A semicolon terminates the statement after `ON CREATE SET r.name = "..."`, making `WITH t` start a new orphaned statement where `t` is undefined.

**Fix:** Remove the semicolon from the `ON CREATE SET r.name = "..."` line so the full Q3+Q4 block remains a single continuous statement.

**Before:**
```cypher
MERGE (t)-[r:HAS_CONTENT]->(c)
ON CREATE SET r.name = "t.edge.CONTENTMENT";
// 3. Pass 't' forward, find the Parent Topic, and link them
WITH t
```

**After:**
```cypher
MERGE (t)-[r:HAS_CONTENT]->(c)
ON CREATE SET r.name = "t.edge.CONTENTMENT"
// 3. Pass 't' forward, find the Parent Topic, and link them
WITH t
```

**Method:** Python script — regex replace `ON CREATE SET r\.name = "([^"]+)";` → `ON CREATE SET r.name = "\1"` within the Cypher block of each of the 7 affected files.

---

### Fix 2 — Populate Empty Parent Names (118 files)

**Problem:** `MATCH (parent:TOPIC {name: ""})` — the parent topic name was never filled in. The correct value already exists in the YAML frontmatter as the `parent:` field.

**Fix:** Read the `parent:` value from each file's YAML frontmatter and inject it into the empty `MATCH (parent:TOPIC {name: ""})` line in the Cypher block.

**Before:**
```cypher
MATCH (parent:TOPIC {name: ""})
```

**After (example from `thought-ANNIHILATION-OF-EVIL.md`):**
```cypher
MATCH (parent:TOPIC {name: "topic.DIVINE SOVEREIGNTY"})
```

**Method:** Python script — for each affected file:
1. Parse the YAML frontmatter to extract the `parent:` value.
2. Replace `MATCH (parent:TOPIC {name: ""})` with `MATCH (parent:TOPIC {name: "<parent_value>"})` in the Cypher block.
3. Write the updated file back.

No manual lookup needed — the correct value is already in every file.

---

### Fix 3 — Correct Typo in Parent Name (1 file)

**Problem:** `tooic.ASTROPHYSICS` is not a valid node name prefix.

**Fix:** Simple find-and-replace in `thought-SUN-ENERGY.md`.

**Before:**

```

**After:**

```
